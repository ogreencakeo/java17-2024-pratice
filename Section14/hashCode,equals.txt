[ hashCode vs equals 메서드 재정의 ]
- 해시코드 : 객체의 내용을 베이스로 만들어진 객체 구분(인식)용 정수 값 -> 식별자
- hasCode() 메서드 변환 타입 -> int 타입을 반환
- equals() 메서드 -> 조상님 클래스인 Object 클래스와 메서드 -> == 연산자 처럼 주소 비교
- == 연산자는 Primitive Type에 대해서는 값 비교가 가능, Reference Type에 대해서는 주소 비교를 수행

[ equals 메서드 재정의(오버라이딩) ]
- equals() 메서드를 통해서 동등 값 비교를 하기 위해서 -> String 클래스는 재정의해서 사용
- 이 말은 우리가 만든 클래스인 경우에도 동등 값 비교를 위해서는 equals() 메서드 재정의가 필요하다는 뜻
