1. 상속이란?
- 우리가 일상에서 자주 듣는 "자식이 부모의 재산을 상속받았다"라는 의미와 같음
- 즉, 부모 클래스가 존재할 때 그 부모 클래스의 자원(속성, 기능)을 재사용(재활용)하여 자식 클래스를 쉽게 만드는 것
- 이러한 것을 "자식 클래스가 부모 클래스의 멤버(필드, 메서드)를 상속받았다"라고 표현
- 단, 접근제어자가 private이면 상속 X
- 상속을 통해서 클래스를 만들게되면 기존의 것을 재사용하는 부분은 구현하지 않아도 되므로 굉장히 효율적!
- 더 나아가 물려받은 유산을 자식이 더 키울 수 있는 것 처럼 자식 클래스가 상속받는 기능에 더 추가해서 만들 수도 있음
- 상속은 클래스 사이에서 이루어지는 것이며, 객체 사이에서는 이루어지지 않는다.

2. 상속 키워드
- extends 키워드 사용
- 단어 자체의 뜻이 "확장하다"인 것 처럼 기존의 클래스를 확장[확대]한다는 개념으로 생각하면 됨
- extends 뒤에 부모 클래스명 하나만 적어주면 됨 -> 자바는 단일 상속만 가능(다중상속 X)

3. 상속 관련 많이 보게되는 용어들
- Parent vs Child
- Supper vs Sub
- 기본 클래스 vs 파생 클래스

4. 상속을 공부할 때는 사고의 발상이 필요
- 만들고자 하는 클래스간의 공통분모를 생각하는게 필요
- 상속이라는 개념을 모를 때는 클래스간 공통분모를 생각하지 못하고 무심코 Fireman, Nurse 클래스를 만들 수 있음.
- 상속을 통해서 만든다면 먼저 공통 분모를 가지는 Person 부모 클래스를 만들어 상속을 시켜주면 됨.
- Fireman, Nurse 자식 클래스는 부모로부터 "사람"의 공통분모를 상속받아 -> 자기 자신만의 속성과 기능을 추가
